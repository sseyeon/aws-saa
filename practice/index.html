문제 JSON 형식 가이드
업로드할 JSON 파일은 아래 구조를 따라야 합니다.
{
  "total": 100,
  "topics": [
    { "name": "S3", "ids": [1, 3, 7] },
    { "name": "EC2", "ids": [2, 4, 5] }
  ],
  "questions": [
    {
      "id": 1,
      "topic": "S3",
      "question": "A company wants to...",
      "question_ko": "한 회사가 ...",
      "options": {
        "A": "Use S3 Transfer Acceleration",
        "B": "Use CloudFront",
        "C": "Use Direct Connect",
        "D": "Use Snowball"
      },
      "options_ko": {
        "A": "S3 Transfer Acceleration 사용",
        "B": "CloudFront 사용",
        "C": "Direct Connect 사용",
        "D": "Snowball 사용"
      },
      "answer": "A",
      "explanation": "S3 Transfer Acceleration...",
      "explanation_ko": "S3 Transfer Acceleration은...",
      "multi_answer": false
    }
  ]
}
필드타입설명
totalnumber전체 문제 수 선택
topicsarray토픽 목록 (name + ids 배열) 선택
questionsarray문제 배열 필수
questions[].idnumber문제 고유 번호 필수
questions[].questionstring영문 문제 필수
questions[].question_kostring한글 문제 선택 (없으면 영문 표시)
questions[].optionsobjectA/B/C/D 선택지 (영문) 필수
questions[].options_koobjectA/B/C/D 선택지 (한글) 선택
questions[].answerstring정답 (예: "A", "BD") 필수
questions[].explanationstring해설 (영문) 선택
questions[].explanation_kostring해설 (한글) 선택
questions[].multi_answerboolean복수 정답 여부 선택 (기본 false)
questions[].topicstring토픽명 선택

최소 요건: questions 배열에 각 문제마다 id, question, options, answer 가 있으면 동작합니다.
한글 필드(question_ko, options_ko)가 없으면 영문으로 표시됩니다.
build_questions.py로 생성한 파일은 이 형식을 자동으로 따릅니다.
