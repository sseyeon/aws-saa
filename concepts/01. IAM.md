# 🔐 AWS IAM 및 액세스 제어(CLI/SDK) 마스터 치트키

## 🎯 1단계: 한 눈에 보는 핵심 키워드 매칭

시험 문제 시나리오에 아래 핵심 단어가 등장하면 고민하지 말고 바로 정답을 매칭하세요!

* **전체 AWS 지역(Region) 및 계정에 공통 적용되는 자격 증명 통제소** $\rightarrow$ `IAM (Identity and Access Management / 전역 서비스)`
* **계정 생성 시 발급되는 무한 권한 자격 증명으로, 실무 및 일상 작업 시 사용 금지 조치 필수** $\rightarrow$ `루트 계정 (Root Account)`
* **사용자가 업무를 수행하는 데 필요한 필수 최소한의 권한만 정밀 할당하는 보안 아키텍처 철학** $\rightarrow$ `최소 권한의 원칙 (Least Privilege Principle)`
* **JSON 정책 문서 내에서 API 접근을 최종 허용할지 차단할지 명시하는 필수 레이어** $\rightarrow$ `Effect (Allow / Deny)`
* **특정 권한이 발동되는 시간, IP 대역, 혹은 암호화 상태 등의 세부 트리거를 지정하는 정책 옵션** $\rightarrow$ `Condition (조건)`
* **사용자가 알고 있는 비밀번호와 소유한 보안 디바이스 토큰을 결합하여 자격 증명 탈취를 원천 방어하는 기술** $\rightarrow$ `MFA (Multi-Factor Authentication)`
* **인터넷 브라우저 콘솔 대신 로컬 터미널 셸(Shell) 환경이나 자동화 스크립트로 AWS 공용 API를 직접 제어하는 도구** $\rightarrow$ `AWS CLI`
* **Java, Python 등의 프로그래밍 언어 라이브러리 형태로 애플리케이션 내에 심어 AWS API를 호출하는 개발 도구** $\rightarrow$ `AWS SDK`
* **사람이 아닌 EC2 인스턴스, Lambda 등의 AWS 리소스에 영구 키 없이 임시 자격 증명을 안전하게 위임하는 개체** $\rightarrow$ `IAM 역할 (IAM Role)`
* **전사 사용자의 MFA 설정 여부, 비밀번호 변경일 등 계정 레벨의 보안 자격 증명 상태를 CSV 일괄 추출하는 감사 도구** $\rightarrow$ `IAM 자격 증명 보고서 (Credentials Report)`
* **사용자 레벨에서 특정 권한을 마지막으로 찌른 시점을 추적하여 안 쓰는 불필요 권한을 솎아내 주는 최소 권한 실현 도구** $\rightarrow$ `IAM 액세스 관리자 (Access Advisor)`

## 📑 2단계: 서비스별 100% 디테일 분석 및 핵심 개념

### 🧑‍💻 1. IAM 사용자(Users), 그룹(Groups) 및 정책 구조

IAM은 리전에 종속되지 않고 AWS 계정 전체에 적용되는 **전역(Global) 서비스**입니다.

[Image showing IAM hierarchy mapping individual physical users into IAM Groups with attached JSON IAM Policies]

* **루트 계정 (Root Account):** 계정 개설 시 자동 생성되는 최상위 계정입니다. **절대 공유하거나 일상 업무에 사용해서는 안 되며**, 마스터 생성 직후 강력한 MFA를 결합한 뒤 봉인하고 개별 IAM 사용자를 파생하여 사용해야 합니다.
* **사용자(User) vs 그룹(Group):** * **User:** 조직 내 실제 물리적인 한 명의 사람(Physical User)과 1:1 매핑되는 독립 개체입니다. AWS 콘솔 접근용 비밀번호 및 API 접근용 액세스 키를 가집니다.
* **Group:** 사용자들의 집합체입니다. **그룹 내에는 오직 사용자(User)만 배치할 수 있으며, 그룹 안에 또 다른 그룹을 중첩 포함(Nested Group)할 수 없습니다.** 사용자를 그룹에 묶어 권한을 부여하는 것이 전사 보안 관리에 매우 유리합니다.



#### 🏛️ IAM 정책(Policy) JSON 내부 스펙 구조

권한은 JSON 포맷의 정책 문서를 통해 정의되며 다음의 6대 핵심 엘리먼트로 구성됩니다.

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "EC2AndELBReadOnly",
            "Effect": "Allow",
            "Principal": "*",
            "Action": [
                "ec2:Describe*",
                "elasticloadbalancing:Describe*"
            ],
            "Resource": "*",
            "Condition": {
                "IpAddress": {"aws:SourceIp": "203.0.113.0/24"}
            }
        }
    ]
}

```

* **Version (필수):** 정책 언어의 규격을 지정합니다. 구형 버전 대신 최신 표준인 **`"2012-10-17"`** 스펙을 사용하는 것이 표준입니다.
* **Id (선택):** 정책 문서 전체를 식별하는 고유 ID입니다.
* **Statement (필수):** 실제 권한 규칙을 담는 배열 컨테이너로, 내부에 다음 항목을 품습니다.
* **Sid (선택):** 해당 Statement 문장을 구별하기 위한 단순 식별자 문자열입니다.
* **Effect (필수):** 특정 API 접근을 허용할지 거부할지 결정합니다. (**`Allow`** / **`Deny`**)
* **Principal (조건부 필수):** 리소스 기반 정책(S3 버킷 정책 등)에서 이 권한을 부여받을 대상(사용자, 계정, 역할)을 명시합니다.
* **Action (필수):** Effect에 의해 허용되거나 거부되는 **AWS API 호출 메서드의 목록**입니다. (예: `ec2:Describe*`)
* **Resource (필수):** 해당 조치(Action)가 적용될 AWS 리소스의 범위(ARN)를 제한합니다. (`*`은 전체 리전 내 모든 자원 의미)
* **Condition (선택):** 이 정책이 활성화되어 적용될 **시간, IP 대역, MFA 인증 여부 등의 세부 조건**을 정밀 통제합니다.

### 🔐 2. 계정 보호 기술: Password Policy & MFA

#### 🔐 강력한 비밀번호 정책 (Password Policy)

계정 보안 강화를 위해 복잡성 규칙을 강제할 수 있습니다.

* 비밀번호 최소 길이 지정 규칙, 특정 문자 구성(대문자, 소문자, 숫자, 특수문자 필수 포함) 요구 기능.
* 일반 IAM 사용자가 자신의 비밀번호를 직접 변경할 수 있도록 허용하거나 금지하는 옵션 제어 가능.
* 일정 기간 경과 시 자격 증명 자동 만료 처리를 통한 주기적 교체 강제 및 **과거 사용한 비밀번호 재사용 금지(기억 횟수 제한)** 설정 지원.

#### 🔐 3가지 MFA(다요소 인증) 디바이스 옵션 명세

비밀번호(알고 있는 것) 외에 보안 디바이스(가지고 있는 것)의 임시 토큰을 결합하여 자격 증명 유출 피해를 원천 차단합니다. 루트 계정과 관리자 권한을 가진 IAM 유저는 도입이 필수적입니다.

[Image displaying multiple AWS MFA authentication methods including virtual app tokens and physical hardware keys]

1. **가상 MFA 장치 (Virtual MFA Device):** 단일 스마트폰 앱 디바이스 안에서 여러 개의 토큰 생성을 지원합니다. 하나의 앱으로 루트 계정, 개별 IAM 사용자, 타 회사 AWS 계정까지 원하는 수만큼 멀티 등록 및 교차 관리가 가능하여 가장 범용적입니다. (예: *Google Authenticator, Authy*)
2. **U2F 보안 키 (Universal 2nd Factor):** 하드웨어 USB/NFC 보안 키 형태입니다. 하나의 물리 보안 키 장치로 다수의 루트 계정 및 IAM 사용자의 로그인 2차 인증을 동시 커버할 수 있습니다. (예: *Yubikey*)
3. **하드웨어 Key Fob 장치 (Hardware Key Fob):** 고정형 일회용 비밀번호(OTP)를 띄워주는 독립형 하드웨어 열쇠고리 장치입니다. 제3자 기업인 Gemalto 사 제품을 주로 활용하며, 특수 망인 AWS GovCloud 전용으로는 SurePassID 사의 전용 하드웨어 톳 장치가 사용됩니다.

### 🖱️ 3. 프로그래밍 방식 접근: AWS CLI, SDK 및 Access Keys

AWS 자원에 접근하는 방법은 크게 3가지 경로로 나뉩니다.

| 🎛️ 접근 방식 유형 | 🔒 보안 메커니즘 인증 수단 | 🛠️ 주요 실무 활용 목적 및 특징 |
| --- | --- | --- |
| **AWS Management Console** | 웹 비밀번호(Password) + MFA | 웹 브라우저 GUI 화면을 통한 시각적 리소스 수동 관리 |
| **AWS CLI (Command Line)** | **액세스 키 (Access Key Pair)** | 콘솔 없이 개발자 PC 터미널 셸에서 명령어로 API 즉시 제어 및 셸 스크립트 기반 작업 자동화 구동 |
| **AWS SDK (Software Dev Kit)** | **액세스 키 (Access Key Pair)** | 프로그래밍 언어별 전용 라이브러리를 애플리케이션 소스 코드 내에 심어 자체적으로 AWS API 통신 수행 |

#### 🔑 액세스 키(Access Keys)의 핵심 취급 주의보

AWS CLI와 SDK 구동을 위해 발급받는 크리덴셜 쌍으로, **일반 웹 로그인 비밀번호와 완전히 동일한 가치**를 지니므로 절대 외부에 공유하거나 소스 코드에 하드코딩해서는 안 됩니다.

* **Access Key ID:** 일반 유저네임(`Username`)과 매칭되는 개념입니다.
* **Secret Access Key:** 유저네임의 패스워드(`Password`)와 매칭되는 개념으로, 최초 발급 시 단 한 번만 다운로드 및 열람이 가능하며 분실 시 재발급이 불가하여 키를 폐기하고 새로 생성해야 합니다.
* **구동 비화:** 프로그래밍 방식으로 구동되는 AWS CLI 도구는 사실상 내부적으로 Python용 AWS 공식 SDK 라이브러리인 **Boto(Boto3)** 아키텍처 위에 구축되어 돌아가는 구조입니다.

### 💡 4. 임시 자격 증명 위임: IAM 역할 (IAM Role)

사람이 아닌 **AWS 서비스(EC2 인스턴스, Lambda 함수, CloudFormation 등)가 타 AWS 자원에 안전하게 접근할 수 있도록 권한을 넘겨주는 개체**입니다.

* **동작 원리:** 개념적으로는 IAM 사용자(User)와 권한 정의 측면에서 유사하지만, **고정된 비밀번호나 영구적인 Access Key 세트를 절대 가지지 않습니다.**
* **임시 토큰 보안:** 역할을 부여받은 EC2 인스턴스나 Lambda는 백엔드에서 AWS STS(보안 토큰 서비스)로부터 주기적으로 갱신되는 단기 임시 자격 증명(Temporary Credentials)을 주입받아 API 호출을 수행합니다. 따라서 자격 증명 탈취 및 하드코딩 소스 코드 유출 리스크로부터 완벽히 자유로워집니다.

### 🧾 5. 2대 IAM 보안 감사 및 거버넌스 도구

기업 인프라의 권한 과잉 상태를 정기적으로 감사하여 '최소 권한 원칙'을 감시하는 도구 세트입니다.

#### 📊 1. IAM 자격 증명 보고서 (Credentials Report)

* **범위:** 계정 레벨 (**Account-level**) 전체 감사 리포트입니다.
* **기능:** 우리 AWS 계정에 존재하는 모든 IAM 사용자의 생성일, 최종 로그인 시점, 웹 비밀번호 마지막 변경일 및 만료 예정일, **MFA 활성화 적용 여부 상태**를 일괄 전수 조사하여 깔끔한 CSV 파일 리포트로 다운로드해 주는 기능입니다.

#### 🧐 2. IAM 액세스 관리자 (Access Advisor)

* **범위:** 개별 사용자 및 그룹 레벨 (**User-level**) 정밀 추적 도구입니다.
* **기능:** 특정 IAM 유저나 역할에게 부여된 수많은 API 권한 목록 중, **해당 유저가 실제로 그 권한을 "마지막으로 호출한 날짜와 시간"을 실시간으로 추적하여 화면에 전시**해 줍니다.
* **실무 활용:** 권한은 잔뜩 받아놓고 지난 90일간 단 한 번도 해당 API를 요청하지 않은 내역이 적발된다면, 불필요한 과잉 권한으로 판단하여 즉시 회수함으로써 시스템을 완전한 **최소 권한 상태**로 다듬어 나갈 수 있습니다.
