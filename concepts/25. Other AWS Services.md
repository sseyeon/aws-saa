# 🛠️ AWS 인프라 자동화 및 커뮤니케이션·유틸리티 서비스 마스터 치트키

## 🎯 1단계: 한 눈에 보는 핵심 키워드 매칭

시험 문제 시나리오에 아래 키워드가 등장하면 즉시 해당 서비스를 매칭하세요!

* **선언적 템플릿(JSON/YAML) + 코드형 인프라(IaC) + 인프라 재생성/예측 프로비저닝** $\rightarrow$ `CloudFormation`
* **CloudFormation에서 시각적으로 리소스 관계를 보며 아키텍처 다이어그램 설계/편집** $\rightarrow$ `CloudFormation Stack Designer`
* **아웃바운드/인바운드 대규모 이메일 엔진 + SMTP/API 연동 + 높은 도메인 신뢰성(DKIM/SPF)** $\rightarrow$ `Amazon SES`
* **이메일/SMS/푸시/음성 마케팅 캠페인 + 사용자 세그먼트 타겟팅 + 양방향 고객 응답 수신** $\rightarrow$ `Amazon Pinpoint`
* **배스천 호스트 / 인바운드 22번 포트 오픈 / SSH Key 없이 보안 셸 브라우저 원격 접속** $\rightarrow$ `SSM Session Manager`
* **여러 EC2 인스턴스에 SSH 연결 없이 동시 스크립트/명령 실행 + 실행 결과 S3/CloudWatch 전송** $\rightarrow$ `SSM Run Command`
* **OS/애플리케이션 보안 패치 자동화 + 특정 일정 및 규정 준수(Compliance) 보고서 생성** $\rightarrow$ `SSM Patch Manager`
* **OS 패치, 드라이버 업데이트 등을 실행할 특정 타임존 일정, 대상, 기간을 미리 정의** $\rightarrow$ `SSM Maintenance Windows`
* **런북(Runbook) 서식을 통한 인스턴스 재시작, AMI 생성 등 복잡한 인프라 반복 작업 자동화** $\rightarrow$ `SSM Automation`
* **최대 12개월 비용 예측 + 월별/시간별/리소스별 비용 세부 분석 + 최적의 Savings Plan 권장** $\rightarrow$ `AWS Cost Explorer`
* **S3에 저장된 미디어 파일을 스마트폰 등 다양한 소비자 재생 장치 전용 포맷으로 인코딩/변환** $\rightarrow$ `Amazon Elastic Transcoder`
* **시간 제한(15분) 없는 대규모 배치 처리 + Docker 이미지 기반 ECS 실행 + EC2/스팟 인스턴스 동적 프로비저닝** $\rightarrow$ `AWS Batch`
* **Salesforce, Slack 등 SaaS 앱과 AWS S3/Redshift 간에 코드 없이(No-Code) 안전한 데이터 전송** $\rightarrow$ `Amazon AppFlow`

## 📑 2단계: 서비스별 100% 디테일 분석 및 핵심 개념

### 🏗️ 1. AWS CloudFormation (코드형 인프라 - IaC)

JSON 또는 YAML 템플릿 파일을 사용해 AWS의 거의 모든 리소스를 코드로 모델링, 프로비저닝, 관리하는 선언적 인프라 자동화 서비스입니다.

#### 💎 핵심 이점 및 비용 최적화 전략

* **선언적 프로비저닝:** "보안 그룹 생성 $\rightarrow$ EC2 2대 바인딩 $\rightarrow$ S3 버킷 생성 $\rightarrow$ ELB 전면 배치"를 템플릿에 명시하면, CloudFormation이 리소스 간의 종속성을 파악하여 올바른 순서로 자동 조율(Orchestration)해 줍니다. 개발자가 생성 순서를 수동 계산할 필요가 없습니다.
* **코드 가이드 거버넌스:** 인프라 변경 시 수동 조작 대신 코드 리뷰(Code Review) 프로세스를 통해 안정성을 검증할 수 있습니다.
* **비용 절감 전략:** 스택 내의 모든 리소스에 식별자 태그(`Tag`)가 자동 지정되므로 비용 추적이 매우 쉽습니다. 특히 **개발(Dev) 환경의 경우 오후 5시에 스택을 통째로 파괴(삭제)하고 다음 날 오전 8시에 템플릿으로 안전하게 재빌드**하는 방식으로 인프라 유지 비용을 극적으로 낮출 수 있습니다.
* **예외 처리:** 거의 모든 AWS 리소스를 지원하지만, 아직 지원되지 않는 특수한 설정이나 리소스의 경우 **사용자 정의 리소스(Custom Resources)** 기능을 트리거하여 보완할 수 있습니다.
* **CloudFormation Stack Designer:** 복잡한 WordPress 스택 같은 인프라 구성을 수동 코딩할 필요 없이, 시각적으로 리소스 간의 연결 관계 다이어그램을 보면서 드래그 앤 드롭 방식으로 템플릿을 디자인하고 편집할 수 있는 도구입니다.

### ✉️ 2. 엔터프라이즈 메시징 서비스 (SES vs Pinpoint)

#### ✉️ Amazon SES (Simple Email Service)

* **대규모 인바운드/아웃바운드 완전 관리형 이메일 전용 엔진**입니다. 애플리케이션 내에서 SMTP 가상 서버를 직접 구축할 필요 없이 AWS 콘솔, API, SMTP 설정을 통해 트랜잭션(비밀번호 재설정, 결제 알림 등) 이메일이나 마케팅 메일을 안전하게 발송합니다.
* **보안 및 신뢰성:** 메일의 스팸 분류를 방지하고 도메인 신뢰성을 높이기 위해 **DKIM**(DomainKeys Identified Mail) 및 **SPF**(Sender Policy Framework) 보안 인증을 기본 지원합니다. 평판 대시보드와 성과 인사이트를 통해 이메일 전달률, 반송률(Bounce), 스팸 피드백 루프 결과를 정밀 통계로 제공합니다. IP 배포 방식도 공유 IP, 전용 IP, 고객 소유 IP(BYOIP) 중 선택 가능합니다.

#### 🎯 Amazon Pinpoint

* **고객 타겟팅 중심의 양방향 옴니채널 마케팅 커뮤니케이션 서비스**입니다. 이메일뿐만 아니라 SMS, 모바일 앱 푸시 알림, 음성 메시지, 인앱(In-app) 메시지까지 통합 지원합니다.
* **💡 SES/SNS와의 결정적 차이점:**
* **SES / SNS:** 개발자가 코드단에서 각 메시지의 발송 대상, 본문 내용, 전송 스케줄러를 하나하나 직접 API로 제어하고 관리해야 합니다.
* **Pinpoint:** 개발 지식 없이도 마케팅 담당자가 직접 콘솔에서 메시지 템플릿을 만들고, 전송 일정을 예약하며, 고도로 개인화된 **고객 세그먼트(Target Segment) 및 전체 마케팅 캠페인 시나리오**를 직접 설계하고 운영할 수 있습니다. 하루 수십억 건의 메시지 스케일 아웃 및 고객의 답장을 수신하는 양방향 커뮤니케이션 기능이 강점입니다.

### 🛡️ 3. AWS Systems Manager (SSM) 핵심 기능 군

#### 🔒 SSM Session Manager (보안 원격 접속)

* **배스천 호스트(Bastion Host) 무력화:** EC2 인스턴스나 온프레미스 서버에 SSH 키패드 없이, 포트 22번을 보안 그룹에서 열지 않고도 AWS 콘솔(웹 브라우저)이나 CLI를 통해 안전한 대화형 셸(Shell) 세션을 시작할 수 있게 합니다.
* **보안성:** SSH 키 분실이나 인바운드 포트 개방에 따른 네트워크 노출 위험이 원천 차단됩니다. 모든 원격 접속 로그 데이터는 감사를 위해 **S3 버킷 또는 CloudWatch Logs로 실시간 스트리밍 전송**되어 저장됩니다. Linux, macOS, Windows를 모두 지원합니다.

#### 📜 SSM Run Command (원격 스크립트 일괄 실행)

* 관리자가 서버에 일일이 로그인(SSH 접근)하지 않고도, SSM 문서(스크립트) 기반의 명령을 **리소스 그룹으로 묶인 수많은 가상 서버에 동시 일괄 실행**하는 기능입니다.
* 명령의 표준 출력/에러 결과물은 AWS 콘솔에 즉시 표시되거나 S3 버킷, CloudWatch Logs로 전송됩니다. IAM 정책과 타이트하게 통합되어 권한을 통제하며, **CloudTrail**을 통해 "누가 어떤 명령을 보냈는지" 투명하게 기록됩니다. 명령의 상태(진행 중, 성공, 실패 등)를 **Amazon SNS** 알림과 연동하거나 **EventBridge**로 자동 트리거할 수 있습니다.

#### 🛠️ SSM Patch Manager (보안 패치 자동화)

* 수많은 EC2 및 온프레미스 서버(Linux, macOS, Windows)의 OS, 애플리케이션, 시스템 보안 업데이트 패치 과정을 자동으로 수행합니다. 인스턴스를 자동 스캔하여 어떤 보안 패치가 누락되었는지 **패치 규정 준수(Compliance) 보고서**를 생성해 주며, 실제 패치 적용은 서비스 영향을 최소화하기 위해 아래의 유지 관리 기간 기능을 연동하여 실행합니다.

#### 📅 SSM Maintenance Windows (유지 관리 기간 스케줄러)

* 운영 중인 비즈니스 인프라에 패치, 드라이버 업데이트, 소프트웨어 설치 등 운영 리스크가 있는 작업을 수행할 "특정 안전 시간대 일정"을 정의하는 도구입니다.
* **4대 구성 요소:** `1) 언제 실행할지(일정)`, `2) 얼마나 오래 열어둘지(기간)`, `3) 어느 서버들에 밀어 넣을지(대상 인스턴스)`, `4) 해당 기간 동안 실제 구동할 SSM 문서/명령(작업)`.

#### 🤖 SSM Automation (인프라 작업 자동화 런북)

* 인스턴스 강제 재시작, 정기 AMI 이미지 생성, EBS 백업 스냅샷 촬영 등 인프라 자원의 반복적인 유지 보수 및 배포 프로세스를 간소화하는 기능입니다.
* **Automation Runbook:** SSM 문서를 기반으로 자동화 단계를 정의하며, AWS가 제공하는 사전 정의 런북이나 사용자 정의 런북을 사용합니다. 수동 실행 외에도 **Amazon EventBridge**, **SSM Maintenance Windows(예약 실행)**, **AWS Config**의 비준수 규칙 리소스를 자동으로 강제 치유(Remediation)하는 연동 메커니즘을 지원합니다.

### 📊 4. AWS Cost Explorer (비용 시각화 및 최적화)

* AWS 리소스 사용량과 비용 흐름을 시간에 따라 직관적으로 시각화하고 분석하는 비용 관리 전용 도구입니다. 전체 마스터 계정의 총합 비용 수준부터 월별, 시간별, 특정 개별 리소스 태그 레벨까지 세부 보고서(Custom Report)를 생성할 수 있습니다.
* **비용 절감 제안:** 사용자의 과거 인프라 소비 패턴을 AI가 머신러닝으로 정밀 분석하여 청구 비용을 가장 크게 낮출 수 있는 **최적의 Savings Plan 계약 옵션을 자동으로 추천**해 줍니다. 과거 트렌드를 기반으로 향후 최대 12개월 뒤까지의 예상 비용(Forecasting)을 시각적으로 예측해 줍니다.

### 🎬 5. Amazon Elastic Transcoder (미디어 포맷 변환)

* Amazon S3 버킷에 저장된 원본 고용량 동영상 및 오디오 미디어 파일을 스마트폰, 태블릿, PC 등 다양한 스펙의 소비자 재생 장치 환경에 맞춰 최적화된 해상도와 미디어 포맷(MP4, FLV 등)으로 **인코딩/트랜스코딩(변환)해 주는 완전 관리형 서비스**입니다.
* **이점:** 대용량 파일과 대규모 변환 요청을 동적으로 처리할 수 있는 높은 확장성을 제공하며, 초기 구축 비용 없이 미디어 변환이 진행된 재생 시간(기간)에 따라서만 비용이 책정되는 매우 효율적인 요금 체계를 가집니다.

---

### 🚀 6. AWS Batch (대규모 배치 컴퓨팅 처리)

작업의 시작과 끝이 명확하게 떨어지는 대규모 일괄 처리(Batch) 컴퓨팅 작업을 최적의 자원으로 자동 스케줄링하여 실행해 주는 완전 관리형 서비스입니다.

#### ⚖️ AWS Batch vs AWS Lambda 비교 분석 (시험 단골 주제)

| 비교 항목 | ⚡ AWS Lambda (서버리스 마이크로) | 🚀 AWS Batch (대규모 중수 대형 배치) |
| --- | --- | --- |
| **실행 시간 한계** | **최대 15분 제한** (시간 초과 시 강제 종료) | **시간 제한 없음** (수 시간, 수 일 동안 가동 가능) |
| **런타임 환경** | AWS가 제공하는 제한된 언어/환경 런타임 | **Docker 이미지** 형태라면 어떠한 런타임/컨텐츠든 무제한 지원 |
| **임시 디스크 공간** | 수십 MB ~ 10GB 수준의 제한된 `/tmp` 공간 | **EBS 볼륨 및 EC2 인스턴스 스토어**를 결합하여 테라바이트급 디스크 확보 가능 |
| **인프라 아키텍처** | 완전한 추상화 서버리스 | 백엔드에서 **Amazon ECS를 통해 EC2 및 Spot 인스턴스**를 동적으로 프로비저닝하여 의존 구동 (인프라 관리는 AWS가 대행) |

* **작동 프로세스:** 개발자는 처리할 로직을 Docker 이미지로 빌드하여 배치 대기열(Queue)에 예약 등록만 해두면 됩니다. AWS Batch가 작업 부하를 감지하여 필요한 CPU/메모리 스펙에 맞춰 비용 최적화된 **스팟 인스턴스(Spot Instance) 또는 EC2**를 알아서 띄우고 작업을 마친 후 즉시 자원을 반납합니다.

### 🌊 7. Amazon AppFlow (코드 없는 SaaS 통합 플랫폼)

개발자가 수많은 사외 SaaS 애플리케이션의 복잡한 API 문서를 공부하고 전용 연동 코드를 한 땀 한 땀 작성할 필요 없이, 몇 번의 클릭(No-Code)만으로 외부 데이터와 AWS 내부 자원을 안전하게 결합하는 완전 관리형 통합 데이터 전송 서비스입니다.

```
[ SaaS 소스 앱 ]                              [ AWS / 사외 타겟 대상 ]
- Salesforce, SAP, Slack,    ──(AppFlow)──>   - Amazon S3, Amazon Redshift
  Zendesk, ServiceNow 등                        - Snowflake, Salesforce 등

```

* **트리거 주기:** 관리자가 지정한 정기 스케줄(일정에 따라), 소스 시스템의 데이터 변경(이벤트 발생에 따라), 혹은 사용자의 즉각적인 요청(On-demand)에 따라 데이터를 펌핑 전송할 수 있습니다.
