# 🌍 Global Infrastructure 마스터 치트키

## 🎯 1단계: 한 눈에 보는 핵심 키워드 매칭

시험 문제 시나리오에 아래 핵심 단어가 등장하면 고민하지 말고 바로 정답을 매칭하세요!

* **전 세계 200개+ 엣지 로케이션, 정적 콘텐츠 캐싱, DDoS 방어, CDN** $\rightarrow$ `CloudFront`
* **S3 버킷을 오리진으로 쓸 때 직접 접근 차단, CloudFront만 접근 허용** $\rightarrow$ `OAC (Origin Access Control)`
* **S3 Origin에서 OAI는** $\rightarrow$ `구식 (Deprecated), OAC가 대세`
* **ALB를 오리진으로 둘 때 EC2를 숨기는 방법** $\rightarrow$ `Public ALB + Private EC2`
* **EC2를 직접 오리진으로 쓸 때** $\rightarrow$ `EC2도 퍼블릭 필요, CloudFront IP 화이트리스트 보안그룹 적용`
* **전 세계 정적 콘텐츠 캐싱(TTL 기반) vs 특정 리전 실시간 영구 복제** $\rightarrow$ `CloudFront vs S3 CRR`
* **IP 기반 국가 차단/허용, 저작권 법적 규제 대응** $\rightarrow$ `CloudFront Geo Restriction`
* **전 세계 전부 사용 / 일부만 / 북미+유럽만 (비용 최저)** $\rightarrow$ `Price Class All / 200 / 100`
* **배포 직후 캐시가 남아 옛날 파일이 보임, TTL 기다리지 않고 즉시 캐시 삭제** $\rightarrow$ `Cache Invalidation`
* **전 세계 동일 Anycast IP 2개 고정, AWS 사설망 경유, 공용 인터넷 홉 최소화** $\rightarrow$ `Global Accelerator`
* **UDP 기반 게임, IoT, VoIP 등 비HTTP 프로토콜 가속** $\rightarrow$ `Global Accelerator`
* **리전 장애 시 1분 내 자동 Failover, 고정 IP 2개 화이트리스트** $\rightarrow$ `Global Accelerator`
* **DNS 캐시 문제 없음, IP가 평생 고정** $\rightarrow$ `Global Accelerator`
* **이미지·동영상 정적 콘텐츠 전 세계 캐싱** $\rightarrow$ `CloudFront`
* **TCP/UDP 가속, 비HTTP, 고정 IP, 1분 내 Failover** $\rightarrow$ `Global Accelerator`

## 📑 2단계: 서비스별 100% 디테일 분석 및 핵심 개념

### 🍿 1. Amazon CloudFront (CDN)

전 세계 주요 도시마다 엣지 로케이션(Edge Location)을 설치해 인기 있는 콘텐츠를 미리 복사(캐싱)해두는 CDN 서비스입니다.

* **Cache Hit:** 엣지 창고에 파일이 있으면 미국 본사를 안 거치고 서울 엣지에서 바로 전달합니다.
* **Cache Miss:** 엣지 창고가 비어있으면 원본(오리진)으로 달려가서 파일을 긁어와 로컬 캐시에 저장한 뒤 유저에게 전달합니다. 다음 유저부터는 캐시에서 빠르게 받아볼 수 있습니다.
* **DDoS 방어:** 트래픽이 전 세계 엣지로 분산되므로 대규모 DDoS 공격에 원본 서버가 터지지 않습니다. 입구에 AWS Shield나 WAF(방화벽)를 붙여 악성 요청을 차단하기 좋은 구조입니다.

#### Origins (원본 소스): 엣지 창고가 비었을 때 어디서 데이터를 가져올지 지정

**① S3 버킷 원본**

* 사진, 영상, 정적 파일(HTML/CSS)을 전 세계에 뿌릴 때 씁니다.
* **OAC (Origin Access Control):** 유저들이 S3 주소로 직접 들어가서 파일을 접근하지 못하도록, S3 문을 완전히 닫아걸고 "오직 CloudFront가 발행한 출입증(OAC)"을 들고 오는 요청만 S3가 열어주도록 철통 보안을 설정합니다. (과거 기술인 OAI는 폐기되고 OAC가 대세!)
* **Ingress (역주행 업로드):** CloudFront의 전 세계 고속도로망을 거꾸로 타고 들어가서 S3 버킷으로 파일을 빠르게 업로드할 때도 쓰입니다.

**② Custom Origin (직접 만든 HTTP 서버 원본)**

* **ALB(로드 밸런서)를 원본으로 둘 때:** CloudFront가 인터넷을 타고 노크해야 하므로 ALB 자체는 **퍼블릭(Public)** 영역에 노출되어야 합니다. 하지만 그 뒤에서 실제로 일하는 EC2 컴퓨터들은 안전하게 **프라이빗(Private)** 서브넷에 숨겨둘 수 있어 실무에서 가장 많이 씁니다.
* **EC2를 직접 원본으로 쓸 때:** 중간에 로드 밸런서가 없기 때문에 CloudFront 엣지가 EC2를 직접 찔러야 합니다. 따라서 **EC2도 퍼블릭** 상태여야 합니다. EC2 보안그룹(SG)에 "오직 AWS가 공식 제공하는 CloudFront IP 목록"만 들어올 수 있게 화이트리스트를 잠가야 합니다.
* **S3 웹사이트:** S3의 '정적 웹사이트 호스팅' 기능을 켠 버킷은 HTTP 주소가 나오는 웹 서버 취급을 받아 커스텀 오리진으로 분류됩니다.

### ⚖️ 2. CloudFront vs S3 교차 리전 복제(CRR) 비교

둘 다 "데이터를 다른 지역으로 보낸다"는 개념이라 시험 함정으로 자주 나옵니다.

| 구분 | CloudFront | S3 CRR (Cross-Region Replication) |
| --- | --- | --- |
| 방식 | TTL 기반 임시 캐싱 | 실시간 영구 복제 |
| 범위 | 전 세계 200개+ 모든 엣지 | 지정한 특정 리전 |
| 파일 수명 | TTL 만료 후 사라짐 | 진짜 파일이 영구 이사 |
| 적합한 용도 | 전 세계 대상 정적 콘텐츠 전송 최적화 | 특정 리전 낮은 지연 시간 동적 데이터, 재해 복구(백업) |

### 🌐 3. Geo Restriction (국가별 차단 자물쇠)

* **원리:** 유저가 접속한 IP 주소를 기반으로 어느 국가인지 판별하는 서드파티 Geo-IP DB를 활용합니다.
* **Allowlist:** "한국, 미국 유저만 들어와!" (지정한 국가만 허용)
* **Blocklist:** "특정 국가 유저들은 차단해!" (지정한 국가만 막음)
* **실무 예시:** 방송 저작권 법률 때문에 "이 영상은 저작권 계약상 특정 국가 안에서만 재생되게 제한해 주세요" 하는 요구사항이 나올 때 무조건 이 기능을 씁니다.

---

### 💰 4. Pricing & Price Classes (비용에 맞춰 엣지 범위 조절)

인프라 회선 비용은 전 세계가 모두 다릅니다. 북미나 유럽은 회선 가격이 저렴하지만, 남미나 아시아 쪽 엣지는 운영비가 훨씬 비쌉니다.

* **Price Class All (요금 계층 전체):** "돈은 상관없으니 전 세계 모든 엣지 다 켜줘!" 전 세계 유저 모두에게 최상의 속도 제공. (비용 최고)
* **Price Class 200:** "대부분 지역은 다 켜되, 남미처럼 제일 비싼 리전 몇 개만 빼줘." (가성비 세팅)
* **Price Class 100:** "예산이 타이트해. 전 세계에서 가장 저렴한 **북미와 유럽 엣지만** 켜줘." (비용 최저, 아시아 유저는 약간의 지연 감수)

### 🔥 5. Cache Invalidations (강제로 캐시 불태우기)

실무에서 배포 직후 "왜 반영이 안 되지?"가 발생할 때 100% 이 문제입니다.

* **문제 상황:** 웹사이트 메인 로고 이미지(`logo.png`)를 새로 수정해서 S3 원본에 올렸지만 유저들에게 옛날 디자인이 계속 보입니다.
* **이유:** 서울 엣지 로케이션 창고가 "나 이 파일 TTL(유효기간) 하루로 설정돼서 내일까지는 본사 안 가고 내 창고에 있는 옛날 파일 그냥 줄 거야" 하고 버티고 있기 때문입니다.
* **해결책 (Invalidation):** TTL 만료까지 기다릴 수 없으니, CloudFront에 "/images/*" 경로에 있는 캐시를 즉시 강제로 삭제(Invalidation) 명령을 내립니다. 그러면 창고가 깨끗이 비워지고, 다음 유저가 올 때 S3 원본에서 따끈따끈한 새 파일을 다시 받아와서 보여주게 됩니다.

### 🏎️ 6. AWS Global Accelerator

**Unicast vs Anycast IP** — Global Accelerator의 마법은 Anycast 주소에서 시작됩니다.

* **유니캐스트(Unicast):** 서버 하나당 고유 IP 하나를 가집니다. 미국 유저도, 한국 유저도 무조건 한국에 있는 그 IP 주소 하나만 바라보고 전 세계 인터넷 망을 헤매며 찾아와야 합니다.
* **애니캐스트(Anycast):** 전 세계 모든 엣지 로케이션이 완전히 동일한 IP 주소를 가집니다. AWS가 딱 2개의 고정 애니캐스트 IP를 주면, 미국 유저가 그 IP를 쳤을 때 미국 엣지로 가고, 한국 유저가 치면 서울 엣지로 자동으로 연결됩니다. 최단 거리 라우팅의 끝판왕입니다.

#### 작동 원리와 핵심 장점

Global Accelerator를 켜면 딱 **2개의 고정 애니캐스트 IP**가 발급됩니다.

1. **지름길 라우팅:** 전 세계 유저가 이 IP로 접속하면 가장 가까운 엣지 로케이션으로 이동합니다. 그 후 엣지에서 진짜 서버(ALB, NLB, EC2 등)까지는 낙후된 공용 인터넷이 아니라 **안정적이고 빠른 AWS 사설 글로벌 네트워크**를 타고 이동합니다.
2. **DNS 캐시 문제 없음:** DNS 기반 라우팅(Route 53)은 서버 주소가 바뀌면 전 세계 통신사 캐시에 옛날 주소가 남아있어 반영되는 데 시간이 걸립니다. 하지만 Global Accelerator는 **IP 주소가 평생 고정**이므로 클라이언트 캐시 문제가 발생하지 않습니다.
3. **지능형 헬스 체크 & 1분 Failover:** 여러 리전 서버들을 감시(Health Check)하다가 한 리전 서버가 폭발하면, **1분 안에 전 세계 트래픽을 살아있는 다른 리전 서버로 자동 우회**시킵니다. 재해 복구(DR) 시나리오의 핵심 치트키입니다.
4. **보안성 향상:** 방화벽 설정 시 외부에는 딱 **2개의 고정 외부 IP만 화이트리스트**로 열어두면 되므로 인프라 문단속이 깔끔합니다. AWS Shield(DDoS 방어)의 보호도 받습니다.

### ⚔️ 7. CloudFront vs Global Accelerator 완벽 비교

둘 다 "전 세계 AWS 엣지 로케이션을 쓴다", "DDoS를 막아준다"는 공통점이 있어서 SAA 시험 단골 함정 카드로 나옵니다.

| 구분 | CloudFront | Global Accelerator |
| --- | --- | --- |
| 핵심 기능 | 캐싱 (Caching) | 프록시 (Proxy) — 캐싱 없음 |
| 프로토콜 | HTTP / HTTPS 위주 | TCP / UDP 전체 지원 (비HTTP 포함) |
| IP | 동적 (바뀔 수 있음) | 고정 Anycast IP 2개 |
| Failover | TTL 기반 (느림) | 1분 내 자동 Failover |
| 적합한 케이스 | 이미지, 동영상, 정적 HTML 전 세계 배포 | UDP 게임, IoT, VoIP, 고정 IP 필요, 빠른 장애 조치 |

* **"창고에 쌓아두고 주느냐(CloudFront)"** vs **"속도 제한 없는 사설 고속도로를 깔아주느냐(Global Accelerator)"** — 이 차이만 기억하면 관련 문제는 10초 만에 풀 수 있습니다.

### 🎯 한눈에 외우는 SAA 시험 정답 매칭 공식

1. **S3 보안 우회 차단, CloudFront만 S3 접근 허용** $\rightarrow$ **OAC**
2. **ALB 뒤 EC2를 숨기기** $\rightarrow$ **Public ALB + Private EC2**
3. **특정 국가 저작권 제어** $\rightarrow$ **Geo Restriction**
4. **가장 저렴한 북미/유럽만 사용** $\rightarrow$ **Price Class 100**
5. **수정사항 즉시 반영, 캐시 강제 삭제** $\rightarrow$ **Cache Invalidation**
6. **해외 유저가 공용 인터넷 홉(Hops) 때문에 너무 느리다** $\rightarrow$ **Global Accelerator**
7. **HTTP가 아닌 UDP 기반 실시간 게임 / IoT 가속** $\rightarrow$ **Global Accelerator**
8. **리전 장애 시 1분 미만의 신속한 재해 복구(Failover)** $\rightarrow$ **Global Accelerator**
9. **전 세계에 딱 2개의 고정 IP(화이트리스트용)만 노출하고 싶다** $\rightarrow$ **Global Accelerator**
10. **이미지나 동영상을 전 세계에 빠르게 뿌리고 싶다(캐싱)** $\rightarrow$ **CloudFront**
